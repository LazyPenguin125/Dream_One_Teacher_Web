# 教師培訓學習平台｜系統開發計劃書（plan.md）

---

## 1. 專案背景與目標

本系統為「教師培訓學習與進度追蹤平台」，用於支援內部教師培訓流程，提供一個可長期營運、可擴充的學習與管理系統。

核心目標如下：
- 建立穩定的教師線上培訓平台
- 支援課程學習、進度追蹤與作業繳交
- 提供後台管理者完整掌握培訓狀況
- 讓非工程人員可直接管理課程內容（CMS）
- 作為未來師資制度、認證與升級制度的基礎系統

本文件作為整個開發團隊（產品、工程、美術、AI 協作工具）的共同依據。

---

## 2. 使用者角色與權限

### 2.1 教師（Teacher）
- 使用 Google 帳號登入系統
- 瀏覽課程與章節內容
- 閱讀文章、觀看影片
- 查看個人學習進度
- 繳交作業（影片或文字）
- 查看管理者回饋

### 2.2 管理者（Admin）
- 管理教師帳號與權限
- 管理課程、章節與內容（CMS）
- 查看所有教師的學習進度
- 檢視與回饋教師作業
- 控制內容顯示與下架狀態

---

## 3. 核心功能模組

### 3.1 登入與身分驗證
- 使用 Supabase Auth
- 支援 Google OAuth 登入
- 登入後取得唯一使用者 ID
- 依角色（Teacher / Admin）控制權限

---

### 3.2 課程與章節系統
- 課程（Course）由多個章節（Lesson）組成
- 每個章節可包含多個內容項目
- 課程與章節可由後台動態新增與調整

---

### 3.3 學習內容系統（文章與影片）
- 內容類型包含：
  - 文章（文字內容）
  - 影片（外部連結，如 YouTube / Vimeo）
- 系統不負責影片實體檔案儲存
- 內容可設定顯示或下架狀態

---

### 3.4 學習進度追蹤
- 每位教師擁有獨立學習進度紀錄
- 進度以章節完成狀態計算
- 以百分比或進度條方式顯示
- 進度資料需跨裝置同步

---

### 3.5 作業繳交系統
- 教師可於指定章節繳交作業
- 作業形式：
  - 影片檔案（上傳至 Supabase Storage）
  - 文字心得（儲存於資料庫）
- 作業資料需關聯：
  - 教師 ID
  - 課程 ID / 章節 ID
  - 繳交時間

---

### 3.6 後台管理系統（Admin Panel）
- 教師列表與帳號狀態
- 學習進度總覽（依教師 / 課程）
- 作業檢視與回饋
- 權限控管（僅管理者可操作）
- 課程、章節、內容管理入口

---

### 3.7 內容管理系統（CMS）

本系統內建簡化型 CMS，僅限管理者使用。

#### 課程與章節管理
- 新增 / 編輯 / 下架課程
- 新增 / 編輯 / 調整章節順序

#### 文章管理
- 新增文章
- 編輯文章標題與內容
- 指定所屬課程與章節
- 設定顯示狀態（草稿 / 發佈 / 下架）

#### 影片管理
- 新增影片（外部連結）
- 編輯影片標題與連結
- 指定所屬課程與章節
- 控制是否顯示於教師端

#### 內容結構與排序
- 可調整章節內內容顯示順序
- 同一章節可同時包含文章與影片

## 4. 系統架構概覽

### 4.1 整體架構說明

使用者（瀏覽器）
↓
前端（React + Tailwind CSS）
↓
Supabase Client SDK / API
↓
Supabase Backend

Auth（登入與權限）

Database（課程、內容、進度、作業）

Storage（作業檔案）

yaml
複製程式碼

---

### 4.2 技術選型

#### 前端
- React
- Tailwind CSS
- 部署平台：Vercel 或 Netlify

#### 後端 / 雲端服務
- Supabase
  - PostgreSQL Database
  - Auth（Google OAuth）
  - Storage（檔案上傳）
  - API 存取

#### 原始碼管理
- GitHub

## 5. 美術與 UI 設計原則

### 5.1 設計核心原則
- 操作直覺、學習成本低
- 以「教育培訓系統」為定位
- 不追求華麗動畫，重視資訊清楚度

---

### 5.2 視覺風格方向
- 整體風格：簡潔、專業、穩定
- 主色調：中性色（白、灰、深藍）
- 強調元素：
  - 學習進度
  - 完成狀態
  - 回饋提示

---

### 5.3 主要畫面規劃
- 登入頁
- 課程列表頁
- 章節閱讀頁
- 個人學習進度頁
- 作業繳交頁
- 管理者後台首頁
- 內容管理頁（文章 / 影片編輯）
